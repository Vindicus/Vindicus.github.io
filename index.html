Hot or Cold

Introduction: This project is a guessing game based on a guess range of 1-100. The game will generate a random number and the user will have to guess the correct number. For example, if the user’s guess is 50 greater or lower than the correct answer, it will display only the ‘ice cold’ css animation.

Difficulties: The hard part was trying to identify the ‘checkpoint’ for each range of numbers in which to display the correct category for either ice cold, cold, warm, hot, or very hot. For example, Taking the absolute value of user input subtracted by the correct answer will yield the difference between the two numbers. If the yielded difference is greater than 10 or less than 20, the ‘ice cold’ css animation will be shown.

The fun part: The most interesting part was creating key frame animations for ice cold, cold, warm, hot, and very hot. I created a key frame named pulse and had it animate infinitely. The proper css animation will display while the other animations will have their opacity set to 0.2 based on the yielded difference from user input.

Youtube API

Introduction: This project is a simple Youtube search app utilizing jQuery’s AJAX function. This app allows users to search any videos and the AJAX will fetch user input and retrieve a set of 20 videos per page. By clicking the ‘next’ button, the app will fetch the next 20 videos.

Difficulties: I have never used the Youtube API before in my life until now. Before I started creating this app, I had to read through Youtube’s API documentation and establish an API key. Although, it took some time but I finally understood what parameters are required to pass through AJAX and what parameters are optional. The other difficult part was figuring out how to fetch the next set of videos when a user clicks the ‘next’ button on the page. Fortunately, I came across a parameter called pageToken for next results and previous results. Every time the ‘next’ or ‘previous’ button is clicked, a new pageToken for the next or previous page is passed through my variable called nextToken or prevToken and will pass it to the AJAX function again to retrieve the next 20 videos.

The fun part: The most interesting part was actually creating the AJAX function and passing in the parameters I needed to create this app such as the video’s title, description, thumbnails, channelTitle, and more. It allowed me to manipulate these data.

CheckMark

Introduction: This project is a simple appointment reminder app. This allows users to create their own appointments and be reminded via phone calls, text messages, or emails. Users can also invite their friends who has a CheckMark account to their appointment. They can also be reminded of the invited invitation via email.

Difficulties: Since I wanted to allow users to receive text messages and phone calls, I decided to try out a free version of the Twilio API. I never used Twilio before so I had to read through documentation and try to render a TWiML for the phone response. In addition, I needed a background process to generate these reminders at specific times so I had to read through the documentation for the gem called Sidekiq. The hardest part of all was trying to create tests. Testing was very difficult for me, I knew ‘what’ to test but it was hard for me to know ‘how’ to test. Although testing is something I lack, I am currently learning to overcome this obstacle.

The fun part: The most interesting part was using the Sendgrid and adding it to the actionmailer to send email reminders. I also used a authentication gem called Devise which was actually very easy to use. I could simply customize messages, fields, and validations.
